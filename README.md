# 데이터 기반 워크 스타일 및 게임 추천 탐색기

## 1. 프로젝트 개요

본 프로젝트는 사용자의 성격 및 업무 스타일을 분석하여, 그에 가장 잘 어울리는 게임 장르와 게임을 추천해주는 심리 검사 웹 애플리케이션입니다.

사용자는 15개의 질문으로 구성된 설문지를 통해 자신의 성향을 응답하며, 시스템은 K-Means 클러스터링 알고리즘을 통해 사용자를 8가지 워크 스타일 중 하나로 분류합니다. 분류된 프로필에 따라, 해당 스타일에 맞는 게임 장르와 실제 게임 목록이 추천됩니다.

## 2. 주요 기능

- **인터랙티브 설문지:** 15개의 질문에 대해 5점 척도로 응답하며, 질문마다 애니메이션 효과와 함께 한 번에 하나씩 제공됩니다.
- **8가지 워크 스타일 프로필:** 응답 결과를 바탕으로 '도전적인 리더', '신중한 분석가' 등 8가지의 독자적인 워크 스타일 프로필 중 하나로 분석됩니다.
- **장르 기반 게임 추천:** 각 프로필에 매칭된 추천 장르가 제시되며, 해당 장르의 게임들이 추천 목록으로 나타납니다.
- **인기 게임 우선 추천:** 추천 게임 목록은 데이터베이스에 플래그된 '인기 게임'을 우선적으로 보여줍니다.
- **한글화된 게임 설명:** 외부 API를 통해 가져온 게임 설명은 한국어로 번역되어 제공됩니다.
- **결과 시각화 및 공유:** 분석된 프로필은 레이더 차트로 시각화되며, 결과 페이지를 다른 사람과 공유할 수 있는 기능이 제공됩니다.

## 3. 기술 스택

- **프론트엔드:** React (TypeScript), Vite, Tailwind CSS, Chart.js
- **백엔드:** Python, FastAPI, Scikit-learn, SQLAlchemy
- **데이터베이스:** PostgreSQL
- **번역:** Deep-Translator

## 4. 실행 방법

### 사전 요구사항

- Python 3.9+
- Node.js 18+
- PostgreSQL

### 백엔드 설정

1.  **디렉토리 이동 및 가상환경 설정**
    ```bash
    cd backend
    python -m venv venv
    source venv/bin/activate  # macOS/Linux
    # venv\Scripts\activate  # Windows
    ```

2.  **의존성 설치**
    ```bash
    pip install -r requirements.txt
    ```

3.  **데이터베이스 설정**
    - PostgreSQL 데이터베이스를 생성합니다.
    - `backend/app/database.py` 파일의 `DATABASE_URL`을 자신의 PostgreSQL 환경에 맞게 수정합니다.

4.  **데이터 처리 및 모델 학습**
    - 아래 스크립트들을 순서대로 실행하여 게임 데이터를 수집/번역하고, 가상 유저 데이터를 생성한 뒤, 클러스터링 모델을 학습시킵니다.
    - **주의:** `ingest_games.py`는 최초 실행 시 약 400개의 게임 설명을 번역하므로 시간이 다소 소요될 수 있습니다.
    ```bash
    python -m app.ingest_games           # 게임 데이터 수집 및 번역
    python -m app.generate_synthetic_data # 가상 유저 데이터 생성
    python -m app.train_model            # K-Means 모델 학습
    python -m app.update_cluster_info    # 클러스터 이름 및 설명 업데이트
    ```

5.  **백엔드 서버 실행**
    ```bash
    uvicorn app.main:app --reload
    ```
    - 서버는 `http://localhost:8000`에서 실행됩니다.

### 프론트엔드 설정

1.  **디렉토리 이동 및 의존성 설치**
    ```bash
    cd frontend
    npm install
    ```

2.  **프론트엔드 개발 서버 실행**
    ```bash
    npm run dev
    ```
    - 애플리케이션은 `http://localhost:5173`에서 확인할 수 있습니다.

## 6. 프로젝트 동기 및 학습 인사이트

### 왜 이 프로젝트를 진행했는가?

- **단순한 심리테스트를 넘어:** 처음에는 MBTI와 같은 간단한 심리테스트 웹 애플리케이션으로 시작했습니다. 하지만 단순히 정해진 로직에 따라 결과를 보여주는 것을 넘어, 실제 **데이터에 기반한 분석과 모델링**에 초점을 맞추고 싶었습니다. 사용자의 응답 데이터를 통해 동적으로 프로필을 생성하고, 의미 있는 추천을 제공하는 시스템을 구축하는 것이 목표였습니다.

- **일과 삶의 연결고리 탐색:** '나의 업무 스타일이 내가 즐기는 게임 스타일과 어떤 관련이 있을까?'라는 재미있는 질문에서 프로젝트가 확장되었습니다. 데이터 과학과 머신러닝 개념을 일상적이고 흥미로운 주제에 적용해보고자 했습니다.

- **End-to-End PoC 경험:** 데이터 수집, 전처리, 모델 학습부터 API 서버 구축, 그리고 인터랙티브 프론트엔드 개발까지, 아이디어를 실제 작동하는 결과물로 만드는 전 과정을 직접 경험하고 싶었습니다.

### 이 프로젝트를 통해 무엇을 배웠는가?

- **비지도 학습의 가능성:** K-Means 클러스터링 같은 비지도 학습 알고리즘이 별도의 정답(label) 없이도 사용자의 응답 데이터만으로 의미 있는 그룹(워크 스타일)을 찾아낼 수 있다는 것을 확인했습니다. 이는 숨겨진 데이터의 패턴을 발견하고, 이를 통해 새로운 가치를 창출할 수 있는 강력한 도구임을 실감하게 했습니다.

- **견고한 추천 시스템의 조건:** 완벽한 추천 시스템이 아니더라도 사용자에게 충분히 유용한 가이드를 제공할 수 있다는 점을 배웠습니다. 초기에는 특정 게임을 직접 추천했지만, 이는 확장성이 떨어지고 데이터에 크게 의존적이었습니다. 이를 **장르 기반 추천 + 인기도** 방식으로 변경함으로써, 더 유연하고 확장 가능한 추천 로직을 구축할 수 있었습니다.

- **Full-Stack 개발의 시너지:**
    - **데이터의 흐름:** 외부 API에서 가져온 원본 데이터(영어 설명, 개발자 노트 등)가 사용자에게 친화적인 정보가 되기까지는 번역, 정제 등 여러 단계의 처리가 필요함을 깨달았습니다.
    - **API 계약의 중요성:** 프론트엔드와 백엔드 간의 데이터 스키마가 일치하지 않아 발생했던 422 오류는, 명확한 API 명세와 규격의 중요성을 다시 한번 일깨워주었습니다.
    - **피드백 기반의 반복적 개선:** "그래프가 안 나와요", "설문지가 불편해요"와 같은 직관적인 피드백이 실제로는 백엔드 데이터베이스의 구조적 문제나 프론트엔드 컴포넌트의 로직 개선으로 이어졌습니다. 이처럼 사용자의 경험을 중심으로 한 반복적인 개발 사이클이 프로젝트의 완성도를 크게 높인다는 것을 체감했습니다.

- **PoC와 기술 부채의 균형:** 초기 단계에서 마이그레이션 툴을 사용하지 않은 빠른 결정이 나중에 `is_popular` 컬럼 추가 시 데이터베이스 스키마 오류로 이어진 경험을 했습니다. 이는 빠른 프로토타이핑 환경에서 발생하는 기술 부채와 그에 따른 트레이드오프를 실제로 경험하는 계기가 되었습니다.
